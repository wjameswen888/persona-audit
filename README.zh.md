# persona-audit

[English](README.md) · **中文** · [日本語](README.ja.md)

**四个假用户，读你给真用户看的文字，告诉你哪句不通。**

你太懂自己的产品，看不见陌生人卡在哪。persona-audit 派四个画像，**只读**你工具输出的文字——不碰代码、不看文档——报告他们读错了什么、被什么吓到、找不到什么。

## 它能抓到什么

拿你 app 每天发给用户的那条简报举例。四个画像*冷读*同一条真实简报——*冷读* = 只看输出、不看代码，像真用户第一次见那样反应：

- **新手：**"上面写'本周：0'，我以为 app 坏了。"
- **老手：**"'净变化'和'总计'都在，我分不清哪个是哪个。"
- **目标用户：**"没有一句告诉我明天为什么还要点开它。"

三个不同角度，撞上同一处困惑。**共识矩阵**——一张按"几个画像各自独立撞上"排序的清单——把这句顶到最前，你在 ship 之前就把它改掉。跨角度的一致、而不是单点挑刺，才是你该照着改的信号。

![persona-audit 产出的共识矩阵——按几个画像各自独立撞上的数量排序](docs/hero-consensus-matrix.zh.png)

> 上面这张图，就是 persona-audit 跑在这份 README 上的结果：四个画像各自独立地指出，一个讲"帮你看见自己盲区"的工具，自己却没有一个 demo。所以现在有了——就是你刚读的那一段。

## 凭什么不是直接让 Claude review 一下就够了

一句"帮我看看"的 prompt，往往顺着你的措辞、说你想听的话。persona-audit 把四个陌生人钉死在固定身份上、谁都不许看你的代码，只把*几个画像各自独立撞上*的东西捞出来——逼出一次顺从的 review 会径直滑过去的盲区。

## 什么时候用

✅ 一个**引擎产出用户向文本**的工具 ship 了新版本/新输出——报告生成器、每日简报、bot 推送文案、读数工具。*（对，一个每天给用户发总结的副业小程序也算。）*
❌ 代码 review · 纯 UI/视觉 QA · 交互叙事 QA。

## 快速上手

```bash
git clone https://github.com/wjameswen888/persona-audit.git
cp -r persona-audit ~/.claude/skills/
```
*（或者把这两行贴给 Claude，让它帮你装。）*

然后直接对 Claude 说：**"对 \<你工具的输出> 跑一次 persona audit。"** 它会凑齐 6–8 份真实输出——你告诉它从哪来（一条演示命令、几条粘贴的样本、或一个临时 dump 脚本）——跑完四个画像，几分钟后把排序好的矩阵交给你。起步零配置：`LOCAL.md` 用来绑定你的产品，但每个钩子都有默认值，不填也能跑。它能读任何语言的输出，跑在你现有的 Claude Code 上——不用额外订阅（它确实会烧 token，跟任何长任务一样）。

## 怎么跑

1. **生成 6–8 份真实输出**——覆盖普通情况、边缘数据、空状态、用户真正看到的那个视图。
2. **四个画像冷读**——镜像（你的真实用户）、新手（黑话+恐慌探测器）、老手（假精度探测器）、以及你刚 ship 的那层的目标用户。
3. **共识矩阵**——按几个画像各自独立撞上同一处的数量排序。"一票"意味着几个角度都觉得别扭——是"值得优先核实"的强提示，不是证据。
4. **ABCD 分层**——**A** 立刻修（错别字、数字错位）· **B** 要做的真 gap · **C** 跟某个刻意的设计选择冲突 → *动手前先问 owner* · **D** 边界外。
5. **别砍清单**——用户真正喜欢的东西，标出来，免得下版本顺手砍了。

## 诚实说它是什么——以及跟"合成用户"有什么不一样

那些工具模拟用户去**做调研**，还常把 LLM 输出包装成"测量"。persona-audit 反过来：它审你**已经在发**的文字，而且对自己的局限直言不讳。四个画像共用同一个底层模型，所以"一票"意味着*几个角度都觉得别扭*，不是统计独立——这是一个单人、零成本、几分钟的**盲区扫描，产出一份待核实的候选清单，不是数据**。照着跨角度的一致去改；当作结论前先核实。

## 安装细节

这是个 [Claude Code skill](https://docs.claude.com/en/docs/claude-code)——按描述里的触发词自动触发（比如 *"persona audit"*、*"冷读审计"*、*"4 画像审计"*）。把 `LOCAL.md.example` 复制成 `LOCAL.md` 来绑定你的产品（已 gitignore；每个钩子都有默认值，不填也能跑）。
